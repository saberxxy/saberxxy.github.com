Linux静默模式安装oracle11g

发表于 2018-02-24 | 分类于 Oracle

引言：在本文详细描述在Linux操作系统中安装Oracle11g数据库，主要分为以下四步：
（1）基础包安装及前期准备
（2）安装Oracle软件
（3）安装监听
（4）安装ORCL实例

  很多时候，oracle是安装在服务器上的Linux环境下。因为服务器的特殊性，一般安装的Linux环境都是不带有图形界面的，这时就要求oracle在命令行模式下进行安装。笔者综合网上的各类oracle静默安装的实例，经过实际安装操作（趟雷踩坑无数），得出了一种可行的静默安装的流程，仅供各位参考。该文章仅针对于oracle 11g R2版本的静默安装，即非图形界面的安装，安装流程如下：

第零步：配置hosts映射文件

  • #vi /etc/hosts
  • 在其中加入“本机IP地址 本机主机名”的配置

第一步：检查软件安装包

  • 通过 rpm -qa | grep 包名的方式检查是否安装了对应的包，如果安装，会显示对应的包的版本号的，如果没有安装，则不会显示任何内容
  • #rpm -qa | grep binutils

  • 如果没有安装。通过yum install 包名 的方式进行安装对应的软件包。
  • #yum install binutils

  • 依次检查如下软件包
  • binutils
  • compat-libcap1
  • compat-libstdc++-33
  • elfutils-libelf
  • elfutils-libelf-devel
  • gcc
  • gcc-c++
  • glibc
  • glibc-devel
  • ksh
  • libgcc
  • libstdc++
  • libaio
  • libaio-devel
  • make
  • sysstat
  • unixODBC
  • unixODBC-devel
  • #yum install binutils compat-libcap1 compat-libstdc++-33 elfutils-libelf elfutils-libelf-devel gcc gcc-c++
  • #yum install glibc glibc-devel ksh libgcc libstdc++ libaio libaio-devel make sysstat unixODBC unixODBC-devel
  • 以上环境是安装oracle必须具备的，若缺少某项环境，可能在安装过程中出现某些未知的错误。

第二步：修改kernel参数

  • #vi /etc/sysctl.conf
  • 在底部添加如下内容
  • net.ipv4.ip_local_port_range = 9000 65500
  • fs.file-max = 6815744
  • kernel.shmall = 10523004
  • kernel.shmmax = 6465333657
  • kernel.shmmni = 4096
  • kernel.sem = 250 32000 100 128
  • net.core.rmem_default=262144
  • net.core.wmem_default=262144
  • net.core.rmem_max=4194304
  • net.core.wmem_max=1048576
  • fs.aio-max-nr = 1048576

  • 让内核参数生效
  • #sysctl –p
  • 修改 linux 内核文件 sysctl -P 报错
  • error: “net.bridge.bridge-nf-call-ip6tables” is an unknown key
  • error: “net.bridge.bridge-nf-call-iptables” is an unknown key
  • error: “net.bridge.bridge-nf-call-arptables” is an unknown key
  • 解决方法如下：
  • #modprobe bridge
  • #lsmod|grep bridge

第三步：创建用户和相应的组

  • #groupadd -g 200 oinstall
  • #groupadd -g 201 dba
  • #useradd -u 440 -g oinstall -G dba -d /home/oracle oracle
  • #passwd oracle

第四步：修改登录系统参数

  • #vi /etc/pam.d/login
  • 在底部添加:
  • session required pam_limits.so

第五步：修改操作系统安全限制等参数

  • #vi /etc/security/limits.conf
  • 在文件底部添加
  • oracle soft nproc 2047
  • oracle hard nproc 16384
  • oracle soft nofile 1024
  • oracle hard nofile 65536

hexo设置指定不渲染的页面

发表于 2018-01-30 | 分类于 hexo

昨天往博客上搬了个程序员老黄历，原地址如下：
http://www.widuu.com/date/index.html

博客的页面如下：

然而建好了之后发现，hexo居然把这个自定义页面按照主题直接渲染了。
于是乎看了配置文件，解决了问题。

在hexo的_config.yml配置文件中，有这么一项配置，只要在skip_render中配置就可以。

  1. 跳过单个文件

    skip_render:

    - “README.md”

  2. 跳过多个文件

    skip_render:

    - “README.md”
    - “CNAME”

  3. 跳过某一类文件

    skip_render:

    - “*.html”

6、进入GitHub项目，添加解析文件，文件内容一行字，即为购买的域名

7、访问http://www.xuexinyu.com，发现已经指向了GitHub托管的博客。如果没有，则需要等待，一般一个小时左右即可。
8、使用hexo，具体方法参见该链接https://www.cnblogs.com/fengxiongZz/p/7707219.html
如果npm下载慢的话，则安装cnpm，用淘宝源，然后再安装诸多依赖文件
http://blog.csdn.net/quuqu/article/details/64121812
9、干掉GitHub上原项目的内容后，一定要记得加CNAME文件，否则域名无法指向那个项目。这个文件是重中之重，项目化成灰也不能忘记加，要不然链接就404了

10、hexo s，启动本地服务，hexo d -g，项目部署。每次修改都先在本地做测试，测试无误之后再发布，否则404了不好收场
11、建立新页面 hexo new post “博客名”，建好的博客在_posts目录下，编辑的话直接使用Sublime就很好，因为编辑的是页面，所以不用每次都重启，直接刷新即可，若刷新没有效果可尝试Ctrl+f5刷新
